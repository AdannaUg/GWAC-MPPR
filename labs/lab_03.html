Lab 3: Collaboration in Git

Goals

  • Learn how to clone a repository.
  • Create and switch to a new branch.
  • Add files, commit changes, and push to your remote branch.
  • Understand the process of making a pull request.

Cloning the Repository

To get started, you will clone the following repository to your local machine:

git clone https://github.com/VincentSD/GWAC-MPPR.git

This command will create a local copy of the repository on your machine.

Navigating to the Submissions Folder

After cloning, navigate into the repository and then into the submissions folder:

Execute:

cd GWAC-MPPR
cd submissions

Creating Your Submission File

Inside the submissions folder, create a text file with your name in the format FirstnameLastname.txt:

Execute:

touch FirstnameLastname.txt

Open this file and add some content, then save and close it.

Creating a New Branch

Now, you will create a new branch for your changes. Use the following command, replacing your-branch-name with a descriptive name:

Execute:

git checkout -b your-branch-name

Adding and Committing Your Changes

Next, you need to add your new file to the staging area and commit your changes:

Execute:

git add FirstnameLastname.txt
git commit -m "Add my submission file"

Pushing to Your Remote Branch

After committing your changes, push your branch to the remote repository:

Execute:

git push origin your-branch-name

Making a Pull Request

Once your branch is pushed, go to the GitHub repository page. You should see a prompt to create a pull request. Click on it, provide a title and description for your PR, then submit it.

Understanding Branching and Merging

Branching allows you to work on features or fixes without affecting the main codebase. After your pull request is reviewed and approved, it can be merged into the main branch, integrating your changes.
