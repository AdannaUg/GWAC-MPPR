Lab 0 Introduction to Git

What is Version Control?

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. It is essential for any team or individual working on software development, document management, or even graphic design.

Think of version control as an organized way to manage multiple versions of your project. Instead of manually saving your files with different names like "project_v1," "project_final," and "project_final_final," version control systems (VCS) allow you to track every change automatically.

Examples of Version Control Systems

  • Local Version Control: This method involves keeping copies of your files on your local system. An example is saving files with different version names manually.
  • Centralized Version Control: In centralized VCS, such as Subversion (SVN) or CVS, a single server contains all versioned files, and clients check out files from this central place.
  • Distributed Version Control: A distributed VCS, such as Git or Mercurial, allows multiple developers to work with their own local repositories and later merge changes back to the main repository. This method provides more flexibility and safety since every copy of the project acts as a backup.

What is Git?

Git is a distributed version control system that allows multiple developers to work on a project simultaneously without interfering with each other's changes. It tracks changes in source code during software development, helping teams collaborate efficiently.

Git is the most popular and widely used version control system today. Its distributed nature means that every developer has a full copy of the project, including its history, which allows for greater collaboration and fewer risks of data loss.

Why Use Git?

  • Version Control: Keep track of changes and revert to previous versions if necessary.
  • Collaboration: Facilitate teamwork by merging changes from different contributors.
  • Branching: Experiment with new features in isolated branches without affecting the main codebase.
  • Backup: Keep your code safe on remote repositories like GitHub or GitLab.

Key Concepts

  • Repository: A directory where your project lives, containing all files and their history.
  • Commit: A snapshot of your project at a certain point in time, including a message describing the changes.
  • Branch: A separate line of development that allows you to work on different features or fixes.
  • Merge: Combining changes from different branches into a single branch.
