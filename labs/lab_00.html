Lab 0 Introduction to Git

What is Git?

Git is a distributed version control system that allows multiple developers to work on a project simultaneously without interfering with each other's changes. It tracks changes in source code during software development, helping teams collaborate efficiently.

Why Use Git?

  • Version Control: Keep track of changes and revert to previous versions if necessary.
  • Collaboration: Facilitate teamwork by merging changes from different contributors.
  • Branching: Experiment with new features in isolated branches without affecting the main codebase.
  • Backup: Keep your code safe on remote repositories like GitHub or GitLab.

Key Concepts

  • Repository: A directory where your project lives, containing all files and their history.
  • Commit: A snapshot of your project at a certain point in time, including a message describing the changes.
  • Branch: A separate line of development that allows you to work on different features or fixes.
  • Merge: Combining changes from different branches into a single branch.
